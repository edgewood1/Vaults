# Pages Router Basics

The Pages Router is the original file-system based router in Next.js, built on top of the `pages/` directory.

## 1. File-System Routing
In the Pages Router, the file name determines the URL.

- **Index Routes**: `pages/index.js` -> `/`
- **Nested Routes**: `pages/blog/index.js` -> `/blog`
- **Static Routes**: `pages/about.js` -> `/about`

### Dynamic Routes
- **Dynamic Segments**: `pages/blog/[slug].js` -> `/blog/hello-world`
  - Access params via the `useRouter` hook or context in data fetching methods.
- **Catch-all Segments**: `pages/docs/[...slug].js` -> `/docs/a/b/c`
  - Matches all subsequent paths.

## 2. Navigation
Use the `<Link>` component for client-side transitions.

```jsx
import Link from 'next/link';

<Link href="/about">About Us</Link>
```

## 3. Data Fetching & Rendering
The Pages Router uses specific functions exported from page files to determine how they are rendered.

### Static Site Generation (SSG)
- **Function**: `getStaticProps`
- **When**: Build time.
- **Use Case**: Marketing pages, blogs, documentation.
- **Dynamic Paths**: Requires `getStaticPaths` to define which dynamic routes to build.

### Server-Side Rendering (SSR)
- **Function**: `getServerSideProps`
- **When**: On every request.
- **Use Case**: Dashboards, personalized data, real-time feeds.

### Incremental Static Regeneration (ISR)
- **Function**: `getStaticProps` with `revalidate` property.
- **When**: Background update after a specific time interval.
- **Use Case**: E-commerce, news sites.

### Client-Side Rendering (CSR)
- **Method**: Standard React `useEffect` or libraries like SWR/React Query.
- **When**: After initial page load.
