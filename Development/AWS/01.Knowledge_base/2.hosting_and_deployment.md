# Hosting, Deployment & DNS

## 1. Hosting Strategies
**A. Static Site (S3 + CloudFront)**
* **S3 Bucket:** Hosts HTML, CSS, JS. Cannot run server-side code.
* **CloudFront:** A Content Delivery Network (CDN). Caches files globally for speed and manages SSL.

**B. Dynamic/Full Stack (Amplify)**
* **Amplify:** A wrapper tool that combines Git, Auth, and Hosting.
* **Workflow:** Connects to GitHub ‚Üí Auto-builds on push ‚Üí Deploys to CloudFront/S3 backend.

**C. Custom Server (EC2)**
* **Manual Setup:** Launch Linux VM ‚Üí Install Nginx/Apache ‚Üí Configure Reverse Proxy.
* **Pros:** Total control. **Cons:** You manage updates and security.

## 2. DNS & Route 53 üåç
**Route 53** acts as both a Domain Registrar and DNS Provider.
* **Hosted Zone:** A container for all DNS records for a domain.
* **Record Types:**
    * **A Record:** Maps domain to IPv4 Address (e.g., `wildwoodrose.org` ‚Üí `44.204.83.90`).
    * **CNAME:** Maps domain to another domain name.
    * **Alias:** Special AWS record pointing to internal resources (like CloudFront) dynamically.
* **NS (Name Server) Record:** Tells the internet *which* servers hold your DNS info.

## 3. CI/CD Pipelines
Automates the "Build and Deploy" process.
* **Source:** GitHub (Triggers on push).
* **Build:** Compiles code (e.g., `npm run build`).
* **Deploy:** Uploads artifacts to S3 or Amplify.
